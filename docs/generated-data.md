# Generated data

AsmKit target data is derived from LLVM target records. The checked-in generated
files are produced by the project's modified TableGen-based generator,
`asmkit-tblgen`.

Building AsmKit does not run LLVM, Python, C++, or `asmkit-tblgen`. The
generated C and header files are normal source inputs for the static library.

## Generated paths

Generated AsmKit tables live under:

- [`src/generated/`](../src/generated/)

Generated public target IDs live under:

- [`include/asmkit/target/`](../include/asmkit/target/)

Generated files include provenance comments and an SPDX marker:

```text
Generated by asmkit-tblgen from LLVM llvmorg-22.1.7 target records.
SPDX-License-Identifier: Apache-2.0 WITH LLVM-exception
```

## What is generated

The generated data includes:

- Decode tables.
- Structured encode tables.
- Instruction metadata.
- Operand schemas.
- Register metadata.
- Predicate metadata.
- Feature metadata.
- CPU model metadata.
- Mnemonic text tables.
- BPF opcode-field literal metadata.

## Modified TableGen pipeline

The generation pipeline consumes LLVM target records and emits AsmKit-specific C
data.

AsmKit overlays add behavior that is not fully represented by LLVM target
records, such as PC-relative fixups, branch-stub selection, detour planning,
and target-specific relocation rejection.
